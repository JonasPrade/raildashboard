# Review-Checkliste

_Keine offene Checkliste._

Diese Datei wird nach jeder substanziellen Änderung (neue Endpoints, neue UI, Schema-/Migrations-Änderungen, neue Features) vom Agent komplett neu geschrieben. Sie enthält dann konkrete Klickpfade und Checks, die du manuell verifizieren musst.

Inhalte zu offenen Fragen, Risiken oder Blockern liegen **nicht** hier — die gehören in `docs/roadmap.md` bzw. `docs/features/feature-<name>.md`.
